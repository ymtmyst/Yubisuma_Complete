## 1. 目的と「最強」の定義

目的はミラー・リバーシ OFF の Complete ルールで最強の AI を作ること。

このゲームは **完全情報・決定論・同時手番・二人ゼロ和のマルコフゲーム** である(`complete_solver/transition.py` で確認済み。乱数なし、隠れ情報なし、同時宣言のみ)。この構造のゲームには一意のゲーム値と定常マルコフ完全均衡(マキシミン戦略)が存在し、各状態の最適混合戦略は「子状態の値を並べた行列ゲームを LP で解く」ことで厳密に求まる。

したがって「最強」を次の3条件で定義する。

1. **非搾取性(近似ナッシュ)**: どんな相手戦略にも期待値で大きく負けない。混合戦略を含む均衡プレイ。
2. **搾取性**: 弱い相手(none/random 等の固定方策、旧モデル)には最短手数で確実に勝つ。
3. **人間妥当性**: 行動分布・初手・スキル使用が人間視点で説明可能(AI_WORKFLOW_RULES の目視ゲート)。

測定方法は §6 の採用ゲートで固定する。勝率単独では判定しない。

## 2. ゼロベース調査で確立した事実(2026-07-12 計測)

計画の根拠となる一次データ。すべて本日実測・実読した。

1. **ルールエンジンは高品質で再利用可能。**
   `complete_solver/`(state / actions / transition / matrix_game / finite_horizon / state_space)は純粋関数・immutable 設計で、132 テストが PASS。視点交代の符号処理(`same_turn_player`)も遷移層で正しく扱われている。**これはプロジェクトの中核資産であり全面的に残す。**

2. **全状態列挙(=全域厳密解)は不可能。**
   ミラー・リバーシ OFF で初期状態から BFS 列挙を実測した結果、8 分で到達 54.1 万状態・未展開フロンティア 43.3 万・拡張 1 回あたり新規状態 4〜7 個のまま減速せず。ストックの部分集合(2^8)× セメント × バフ類 × 両者分の組合せで、到達可能空間は少なくとも 10^7 オーダー以上と推定される。**「小さいゲームだから値反復で終わり」は成立しない。** 旧計画の `max_states=400〜500` の値反復は全体のごく一部を leaf=0 で打ち切った近似であり、そこから作った BC 教師(zero leaf)が数字偏重の根本原因だったことと整合する。

3. **深さ制限 LP 探索は既に存在するが遅い。**
   `FiniteHorizonSolver` は「LP バックアップ付き深さ制限探索」そのもの(=本計画の実戦エンジンの原型)。ただし素の Python で初期局面(TP 45 手 × NTP 9 手)の depth=3 が 15.7 秒、depth=4 が 121 秒。**高速化が第一級の課題。**

4. **GPU 環境あり。** Python 3.10 / torch 2.6.0+cu124 / CUDA 利用可。価値ネット学習は問題なし。

5. **旧路線(BC + MaskablePPO)の失敗は偶発ではなく構造的。**
   - zero-leaf VI 教師 → 数字 100% 偏重(境界アーティファクト)
   - 反応履歴 16 次元の観測 → 状態が非マルコフ化し、均衡概念と評価指標(exploitability)が壊れた
   - 決定論的初手(フェイント固定・スキップ固定)→ 同時手番ゲームでは決定的方策は原理的に搾取可能。ent_coef 調整では解決しなかった(item 42)
   - none 相手の無限ループ打ち切り → モデルフリー RL が「勝ち筋の発見」を報酬信号だけで行う限界
   これらは PPO の調整不足ではなく、**モデルフリー・方策ネット中心のアプローチがこのゲームの既知構造(完全情報・厳密遷移・LP で解ける局所均衡)を捨てていること**に起因する。

## 3. 新方針: 探索中心アーキテクチャ(LP バックアップ探索 + 価値ネット + 終盤厳密テーブル)

AlphaZero の構図を同時手番ゼロ和ゲーム用に置き換えたもの。方策ネットを主役にせず、**探索を主役**にする。

### 実戦時(プレイ)

```
着手 = 深さ制限探索(depth 2〜4, 適応)
        ├─ 各ノード: 子の値を並べた行列ゲームを LP で解き、値と混合戦略を得る
        ├─ 葉: 学習済み価値ネット V(s)(107 次元特徴量)
        ├─ 終盤・低複雑度状態: 厳密テーブルの値を直接参照
        └─ ルートの LP 混合戦略から実際に「サンプリング」して着手(決定論にしない)
```

- **混合戦略サンプリングが仕様の中心。** 同時手番ゲームの均衡は本質的に混合であり、これが旧モデルの「初手固定を突かれる」問題を原理レベルで消す。
- none のような受動的相手には探索が数手先の強制勝ちを直接発見する。γ<1 により最短勝利が選好されるため、無限ループ打ち切りは起こり得ない。

### 学習時(fitted Nash value iteration)

```
繰り返し:
  1. 自己対戦で状態を収集(双方が現行探索方策、LP 混合からサンプル + ε ランダム注入)
  2. 収集した各状態に「depth-d LP バックアップ値(葉=現行ネット)」を教師値として付与
  3. 終端値・終盤厳密テーブル値をアンカーとして混ぜ、V(s) を回帰学習
  4. 新旧ネットで世代対戦・テーブル一致度を測り、改善していれば採用
```

これはマルコフゲームに対する fitted Nash-VI(minimax-Q の関数近似版)であり、完全情報ゼロ和では値が一意なので、方策の非定常性・忘却(PPO で起きた catastrophic forgetting)の問題構造がそもそも生じない。**学習対象は「値」だけ**で、方策は常に探索+LP が現場で導出する。

### 旧路線の問題がどう消えるか(対応表)

| 旧問題 | 本アーキテクチャでの帰結 |
|---|---|
| zero-leaf 教師の数字偏重 | 葉は学習済み V + 終端/テーブルアンカー。leaf=0 境界が存在しない |
| 初手決定論(フェイント/スキップ固定) | LP 混合戦略からサンプリング。均衡混合が仕様として保証 |
| none 相手の無限ループ | 探索が強制勝ちを直接読む。γ<1 で最短勝利選好 |
| 反応履歴で非マルコフ化 | 状態は完全観測でマルコフ。履歴特徴量は不要(廃止) |
| exploitability 指標の破綻 | BR 攻撃エージェントによる実測搾取率に置換(§6) |
| PPO の視点符号バグ類 | solver の遷移層(検証済みコード)だけを使う |

## 6. 採用ゲート(最終合格条件)

1. **固定方策制圧**: none / counter_first / block_first / random / 分離系(none_uniform 等)すべてに勝率 ≥ 99%、打ち切り 0(none 含む)。
2. **旧世代超え**: 旧ベスト PPO(`maskable_ppo_bc_standard_episode_mixed`、`maskable_ppo_bc_depth4_standard_proper`)との直接対戦で勝率 ≥ 70%。
3. **実測非搾取性**: 凍結した本エージェントに対し BR-PPO 攻撃者(250k steps × 3 seeds)を訓練しても、本エージェント側勝率 ≥ 55% を維持。
4. **人間妥当性**: 初手分布・スキル使用率・開始側手順レポートで警告 0 + ユーザー目視承認。
5. 上記すべてを **開始側/後手側の両視点**で集計する(旧レポートの視点混同バグの教訓)。

## 7. リスクと対策

| リスク | 対策 |
|---|---|
| N1 の速度目標未達 | ダブルオラクル・キャッシュ・並列自己対戦 → それでも不足なら Numba/Rust コア化を N1 末で即断(計画の律速なので先送りしない) |
| 価値ネットがストック組合せに汎化しない | ストックはスキル別 bit で特徴量化済み。N2 テーブルをアンカー・検証基準に常用。ランダム局面注入で分布拡張 |
| ブートストラップ学習の発散 | 終端値+テーブルの固定アンカー、ターゲットネット、世代ゲートで退行検出したら停止・報告 |
| LP 縮退・タイ | 既存 tolerance 処理を維持。値タイは「短手数勝ち優先」でタイブレーク |
| γ の恣意性 | 旧 gamma sweep で感度小を確認済み。N7 で 0.997/0.999 比較を最終確認 |
| 「探索が読み切る前提」の穴(超長期戦術) | depth 不足分は V が吸収する設計。N6 の BR 攻撃が最終的な安全網 |
