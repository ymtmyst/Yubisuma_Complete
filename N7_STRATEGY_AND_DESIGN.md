作成日: 2026-07-14
位置づけ: 深さ4教師に着手する前に、「単に探索深さを彫る」以外の強化路線を再検討した記録と、
採用する3路線(A/B/C)の設計案。マスタープランは §5 N7 に本書の要約を反映済み。

## Part 0. TL;DR

- **アルゴリズムの「族」(LP バックアップ探索 + 価値ネット + fitted Nash-VI)は変えない。** これは
  AlphaZero を「同時手番ゼロ和」に正しく一般化した先にいるものであり、すでに正解。MuZero / CFR /
  PSRO への乗り換えは、本プロジェクトの真のボトルネック(長期スキルの信用割当)を解かないうえ、
  完全情報+厳密シミュレータという強みを捨てるので不採用。
- **「深さ4を掘る」は主戦略にしない。** 遅効性スキル(セメント/ストック/ロック)が報われるのは
  約10プライ先で、深さ4(葉+2プライ)では構造的に届かない。深さは指数コストで、投資の上限が見える。
- **本当のレバーは「価値ネットの長期精度」と「訓練分布のカバレッジ」。** 価値ネットが完璧なら深さ1で
  十分 ── 深さ探索は「ネットが表現しきれない地平線の先」の穴埋めにすぎない。ネット精度への投資は
  原理的に無限の地平線を買える。
- 採用する3路線:
  - **A(本命): 到達可能サブグラフ Nash 値反復** — 決定論+完全情報を使い切り、深さを掘らずに
    長期価値をサブグラフ全域に伝播させて教師値を作る。A0 厳密DBの機構(`packed_vi`)の一般化。
  - **B: 価値ネットの構成エンコーダ化** — フラット特徴では潰れる「非線形スキルコンボ」を、
    構成(保持スキル集合/盤面)への集合・注意エンコーダで表現。
  - **C: 選択的/最良優先探索** — 均一深さの代わりに均衡台(support)上の重要ラインだけ深掘り。
    同コストで実効深さ6〜8。
  - **D(並行・期待は持たない): 深さ4教師** — インフラ完成済みなので安価に回して確認材料にする。

## Part 1. 考察(なぜこの結論か)

### 1-1. 今のアーキテクチャは「AlphaZero」ではない(すでにその先)

現行 Complete AI は **LP バックアップ深さ制限探索 + 価値ネット葉評価 + fitted Nash-VI 自己対戦**。
AlphaZero は MCTS(PUCT)で **交互手番** を前提に子の `max` でバックアップする。ユビスマは
**同時手番** なので各ノードで `max` ではなく **行列ゲームの均衡(LP)** を解く必要があり、本 AI は
それを実装している(`complete_ai/batched_search.py` の `_backup`/`_matrix_value`)。
→ **AlphaZero を同時手番ゼロ和向けに正しく一般化した位置**にすでにいる。

平易版: 「将棋型AI(AlphaZero)」は交互に指す前提の道具。ユビスマは同時に手を出すので、そのままでは
使えず、我々は「毎ノードでじゃんけんの最適配分を計算する版」に作り替えてある。もう普通の AlphaZero
の外にいる。

### 1-2. 他アーキテクチャに乗り換える価値は低い

| 候補 | 判定 | 理由 |
|---|---|---|
| MuZero | 不採用 | 学習ダイナミクスで「シミュレータが無い」問題を埋める技術。我々は完全決定論の厳密 solver を持つので買うものがなく、純粋な精度劣化。 |
| CFR / Deep CFR | 不採用 | 不完全情報ゲーム用。ユビスマは完全情報でオーバーキル。 |
| PSRO / NFSP | 見送り | exploitability を直接叩くのは魅力だが、真の課題(長期地平線)を解かない。N6 で非搾取性は実測クリア済みで優先度が低い。 |

→ **族は維持**。変えるのは「価値をどう伝播させ、どう表現するか」。

### 1-3. 深さ4を主戦略にしない理由

ドメイン知識(ユーザー由来のリトマス指標)より:
- セメントは約10ターン後に効く遅効性スキル。「深さが10級に伸びればストック・セメントの価値は
  大幅上昇するはず」。
- 深さ4は葉が2プライ伸びるだけ。約10プライ先の効果には構造的に届かない。指数コストで10プライは不可能。

→ 深さは「安いので回して確認する」価値はある(インフラ完成済み・1世代~31秒)が、**これで長期スキルが
開花すると期待するのは無理筋**。

### 1-4. 真のボトルネック = ネットの長期精度 + カバレッジ(鶏と卵)

このアーキテクチャの心臓は価値ネット。理屈上 **ネットが完璧なら深さ1で十分**(葉=地平線の先の真の値)。
逆に深さ探索は「ネットが表現しきれない分の穴埋め」でしかない。**深さへの投資は上限が見えるが、ネット精度
への投資は原理的に無限の地平線を買える。**

なぜ長期スキルが学ばれないか ── **鶏と卵**:
現在の(近視眼的)方策で自己対戦 → セメント/ストックが報われる局面に到達しない → ネットが価値を学ばない
→ 探索がそのスキルを打たない → 永遠に到達しない。
これは深さの問題ではなく **訓練分布のカバレッジ問題**。恣意的な強制は禁止(ドメインメモ準拠)だが、
中立な手段(ランダム開幕・ε・広域到達状態サンプリング)でカバレッジを広げるのは既許可路線の延長。

## Part 2. 設計案 A(本命)— 到達可能サブグラフ Nash 値反復

### A-0. 一言で

深さ制限探索は「ルートから前向きに d プライ」しか見ない。代わりに、**到達可能状態を大量にサンプリングして
1枚の巨大なゲームグラフ(DAG に近い有向グラフ)を作り、その上で Nash 値反復を回して収束値を得る**。
収束値はグラフの端から端まで(長い連鎖では20〜40プライ)価値を伝播させたものなので、**深さを掘らずに
長期地平線を稼ぐ**。この収束値を価値ネットの教師にすれば、ネットは「深さ4では絶対見えない長期価値」を
直接学べる。

平易版: いま AI は「ルートから数手先まで」しか読んでいない。代わりに、あちこちの局面を大量に集めて
巨大な「すごろく盤の地図」を作り、地図全体で「勝ちに近い/遠い」をじわじわ染み込ませる。染み込んだ数字を
お手本にすればネットが長期の価値を覚えられる、という作戦。

### A-1. なぜ実現可能か(機構はすでに実証済み)

`complete_solver/packed_vi.py` に、まさにこの計算の完成品がある。ただし今は
**A0(ストック禁止・閉じた宇宙)専用**:

- `_enumerate(root0, root1, alphabet_mask, max_stock, cap)` — 閉集合を njit BFS で列挙
- `_count_cells` / `_fill_tables` — 各状態の子行列(TP×NTP)を CSR 的に平坦化
- `_jacobi_sweep(...)` — 全状態を一斉更新するヤコビ式 Nash 値反復(GS は汚染値の自己維持振動を
  起こすためヤコビに切替済み ── この教訓がそのまま効く)
- `_matrix_value` / `_simplex_value` — 各セルの行列ゲーム値(縮退時は摂動再試行)
- `PackedEndgameDB` / `solve_universe(...)` — 収束値の格納・保存・プローブ

**実績**: A0 = 33.4万状態が **54スイープ25秒で収束**。つまり「状態集合上のヤコビ Nash-VI」は
規模・収束・数値安定性すべて検証済み。A はこれを **閉じた宇宙 → サンプリングしたサブグラフ** に一般化する
だけで、新規性は「境界(フロンティア)の閉じ方」に集約される。

### A-2. A0 との本質的な違い = フロンティアの扱い

A0 宇宙は「ストック禁止」という制約で **閉じている**(全ての子も宇宙内)。一方サンプリングした
サブグラフは閉じていない ── 端の状態(フロンティア)の子は enumerate 対象外。ここが唯一の新規設計:

- **内部状態(interior)**: 子がすべてサブグラフ内 → 通常のヤコビ更新。
- **フロンティア状態(frontier)**: 子が未列挙 → その値を **価値ネット V(s)**(＋ A0/厳密DBに在れば
  そちらを優先)で **固定境界条件** として与える。
- 終端状態: 既存どおり厳密終端値で固定。

これは数学的には「フロンティアを境界としたグラフ上の fitted value iteration」で、深さ制限探索の
「葉=ネット」を **面(サブグラフ全域)** に広げた形。深さ d 探索が「ルートから d 段の木」なのに対し、
A は「サンプリング領域を覆う DAG」で、長い連鎖に沿って価値がずっと奥まで伝播する。

### A-3. 状態サンプリング(カバレッジの中立な広げ方)

グラフに入れる状態集合 S を、恣意性なく広く取る(ドメインメモの禁止事項に抵触しない中立手段のみ):

1. **自己対戦の訪問状態**(既存 `selfplay.run_selfplay` の収集をそのまま流用) ── 実戦分布。
2. **初期局面からの制限付き BFS**(`packed_engine.step` + `legal_tp_codes`/`legal_ntp_codes`) ──
   `cap` と `max_stock`(ドメイン知識: ストックは基本 2、例外 3、4個以上は不要)で規模制御。
3. **ランダム開幕/ランダムウォーク種**(既存 ε・0-2手ランダムの思想の延長) ── 長期スキルが絡む
   到達領域へ中立に踏み込む。特にセメント/ストックを含む局面を「打たせる強制」ではなく
   「そこを地図に載せる」形で確保する(価値の書き換えは VI が担う=恣意性なし)。
4. **A0/厳密DBの状態を境界アンカーとして常時合流** ── 端が厳密値で閉じるほど内部の収束値も正確。

規模の目安: まず 10^5〜10^6 状態で試し(A0 実績から数十秒〜数分で収束見込み)、収束品質と
教師寄与を見て 10^7 に拡大するか判断。**silent cap は付けない**(切り詰めたら log とレポートに明記)。

### A-4. 教師としての使い方(fitted Nash-VI への接続)

現行の世代ループ(`generation_loop` → `dataset` → `train_on`)の教師値生成を、深さ3/4 バックアップから
**グラフVI収束値**に差し替え(または併用):

```
世代ごと:
  1. 状態集合 S を収集(A-3 の 1〜4)
  2. サブグラフ構築(子展開・平坦化: _count_cells/_fill_tables を S 向けに一般化)
  3. フロンティア値 = 現行ネット V(s)(＋厳密DB) を注入
  4. ヤコビ Nash-VI を収束まで(_jacobi_sweep 再利用、GSは使わない)
  5. 収束値 v*(s) を S 上の教師として回帰学習(train_on)
  6. 世代対戦・A0相関・分布レポートで採否判定(既存ゲート)
```

深さ3教師との違い: 深さ3は各状態を独立に「3プライ木」で評価するが、A は S 全体を **連立** で解くので、
S 内で共有される長期構造(セメントが効く筋、ストックの2個型の価値)が **相互に整合した値** として
染み出す。これがリトマス「序盤セメント > 終盤セメント」の逆転を狙う本命。

### A-5. 実装タスク分解(粒度: 実装可能)

| # | 内容 | 再利用 | 新規 | 目安 |

## Part 3. 設計案 B — 価値ネットの構成エンコーダ化

### B-0. 一言で

現行ネットは 103〜107 次元のフラット特徴 MLP。ドメイン知識では「スキル/ストックの組合せは非線形コンボ」
かつ「特定スキル同士に強力なコンボが存在」。フラット結合では組合せが原理的に潰れやすい。**保持スキル集合や
盤面構成を、集合/注意(self-attention)エンコーダで表現**し、コンボを明示的に捉えられる形にする。

### B-1. 設計方針

- 入力を「フラットな数値ベクトル」から「**トークン集合**」に再構成:
  - 各プレイヤーの保持スキル(ストック内容)を1トークンずつ(スキル種別 embedding + 属性)
  - 盤面グローバル特徴(手数・タイム・セメント状態・フェーズ)を CLS トークンに集約
- 小さな **self-attention(1〜2層)** でトークン間相互作用を学習 → コンボ(例: 対カウンター系×通常、
  ロック×タイム)を1層で表現。出力は CLS からスカラー値 V(s)。
- ドメイン制約を構造に反映: ストックは最大 3 トークン(4個以上は不要)、順序不変(集合)なので
  permutation-invariant なエンコーダが自然。

### B-2. 検証(既存ものさしを流用)

- A0 厳密スライスでの MSE / 順位相関(現行の 0.919 / 深さ3相関 0.827 を基準に、
  **ストックを含む局面での相関**を分離集計 ── ここが伸びれば B の効き目)。
- material / 現行フラットネットとの直接ものさし比較(N3 のレポート様式を流用)。
- 過学習・推論速度(探索の葉評価で毎ノード呼ぶので、注意層のコストが探索速度を殺さないか実測)。

### B-3. リスク

- 注意層は葉評価のホットパス。**推論が重くなると探索スループットが落ちる** → まず 1 層・小 d_model で、
  速度退行が閾値内かを最初に確認(ダメなら蒸留 or 特徴量追加に留める)。
- データが足りないと注意は過学習。A(グラフVI)で教師が厚くなってから B に載せる相性が良い。

## Part 4. 設計案 C — 選択的/最良優先探索(smarter, not deeper)

### C-0. 一言で

均一な深さ4ではなく、**均衡混合戦略の台(support)上の手だけを選択的に深掘り**する反復深化。
大半の手は均衡でほぼ0確率なので展開不要。重要ラインだけ実効深さ6〜8まで伸ばせば、深さ4と同コストで
より深く読める。

### C-1. 設計方針

- ルートで浅く LP を解き、**support(確率>閾値の手)** を得る。
- support 上の子だけを1段深く再展開 → 再び LP。support 外は葉(ネット値)で据え置き。
- 反復深化で予算(時間/展開数)まで support を掘り下げる。同時手番なので単一 PV はないが、
  「support の直積」を掘る形で近似する。
- 着手(深さ2据置の現行方針)と教師(深さ3/4 or A)双方に効く。特に **実戦エージェントの手番**で
  重要局面だけ深く読めるのは体感の強さに直結。

### C-2. 検証

- 同一計算予算で「均一深さ4」対「選択的深さ(実効6〜8)」の値精度(A0/厳密比較)と対戦勝率。
- support 閾値の感度(小さすぎると木が爆発、大きすぎると近視眼)。
- 探索スループット(展開数/秒)と着手品質のトレードオフ曲線。

### C-3. リスク

- 同時手番で support の直積を掘ると木が広がりやすい → 閾値と幅制限(top-k support)で抑制、log 明記。
- support が均衡の数値誤差で不安定 → 既存の tolerance/摂動処理を踏襲。

## Part 5. 実施順序・依存関係

推奨順序と依存:

1. **A(本命)を先に**。理由: 教師の質を根本から底上げする=B/C/D すべての土台。A で長期価値が
   ネットに入り始めれば、リトマス(序盤セメント>終盤)の初動が見えるはず。
2. **C を A と並行可**(独立性が高い。探索側の改良で、教師生成にも実戦にも効く)。
3. **B は A の後**が相性最良(厚い教師ができてから表現力を上げると過学習しにくい)。
4. **D(深さ4)は安価なので随時**。A の効果測定の対照(「深さでは届かない」ことの実証)にも使える。

## Part 6. 本質的な強化策の考察(2026-07-14 追記、ユーザー問い「深さはどこまで/深さ以外は」)

前提の実測: ゲームは平均約9手で終局・手の合計は単調減少(層構造)・selective のコストは深さ+1で約3〜4倍
(56→232→639セル/局面)・ネットは教師を丸暗記(val_mse 0.0005)=律速は教師の質。

### 6-1. 深さの到達可能上限
3〜4倍/手の外挿: 深さ7≈6千セル、深さ10≈17万セル/局面。njit コンパイル(本プロジェクト実績 50〜1000倍)後は
**深さ10で~1s/手 = 実用圏**。そして平均ゲーム長が9手なので、**深さ10 ≈ 読み切り**(葉が終局値=誤差源消滅)。
つまり深さは無限のレースではなく「ゲーム長に届いたら終わり」のゴールがある。リスク: support 見落とし
(浅読みで悪く見える手を永遠に掘らない)→ 深さ予算に応じた閾値緩和・top-k 強制深掘り・厳密ソルバ照合で対処。

### 6-2. 深さ以外の本質策(優先順)
1. **層の挟み撃ち(本命)**: 手の合計が単調減少 → (1,1)終盤層を現実的ストック制限(s2/s3、ladder で誤差測定可能)
   で厳密に解き(A0/s1_A4/s2_A4 は N2b で解済み)、前からは selective 深さ6前後で層境界に接続。
   **到達点=「均衡 support 沿いの弱解決」= 神視点の最適解**。ゲームの短さと層構造ゆえ現実的射程。
2. **graph-BR(真の exploitability 計)**: 凍結エージェント相手の攻撃者問題は MDP → subgraph_vi の片側固定版で
   厳密に近い BR が計算可能。PPO 攻撃者(弱い下界)を置き換える「神視点への距離」の計器。
3. **教師の質ループ**: コンパイル済み selective d6-8 を教師に(ネットは丸暗記できるので教師精度=ネット精度)。
   graph-VI は教師平滑化として併用(実証済み)。

### 6-3. 本質的でないと確定したもの(証拠付き)
- B(ネット大型化/構成エンコーダ): 教師丸暗記済み=ネット容量は律速でない(val_mse 0.0005)
- 同一教師で世代数増: 12世代で頭打ち実測
- PPO 攻撃者の強化: graph-BR に本質で劣る

### 6-3b. 重要な caveat: ゲーム長は方策依存(ユーザー指摘 2026-07-14)
「平均9〜10手」は**セメント/ストック/ロックを使わない現行方策**での測定値。これらが真に有効なら(特にストックは
ゲームを延ばしうる)最適プレイのゲーム長は伸び、必要深さの前提が変わる。=「深さ=ゲーム長で読み切り」の上限は
下界にすぎない。ただし長期スキルは間接的・使用が難しく学習難度が高い(真に弱い可能性も残る)。**含意**: 深さ一本足
より、ゲーム長に依存しない**層の挟み撃ち**と**ストック込み厳密終盤テーブル(s2/s3)**の価値が上がる。深さ外挿は
保守的に見積もる。

### 6-3c. レジーム切替仮説とその定量確認(2026-07-14、ユーザー洞察)
ユーザー指摘: タイム(次いでロック/セメント/ストック/クイック/チャージ)は**ゲーム構造を離散的に変える**
(タイム中は連続ターン系が減価しロック/セメントが増価、相手ストックは早期決着圧、勝敗確定盤面が増える等)
= フラグ×スキルの相互作用項(レジーム切替)。**A0厳密スライスをレジーム別に分割して定量確認**
(`complete_ai/n7_regime_slices.py`、パック bit15=time_active):
- タイム発動中 39,177件: 順位相関 0.844(良)だが**絶対誤差 0.150 vs 非発動 0.114(+32%悪化)**。両ネット
  (gvi/depth)で一貫。アルティメット未使用(参入判断局面)も悪い(0.139-0.146)。
- **含意**: LP行列は絶対値同士の比較なので、レジーム較正の歪み=参入判断の系統的歪み。タイム使用0%→レジーム
  データ暗黒→較正悪化→参入判断歪む→使わない、の**レジーム版鶏卵**。ロック/セメントの条件付き価値(タイム中
  増価)も暗黒の中=見かけの弱さの一部を説明。
- **戦略的最重要**: 相手はタイムを使える → 非搾取性にはレジーム対応力が必須。PPO攻撃者はレジーム搾取を
  見つけるには弱い可能性 → 現行 exploitability は楽観的かも(graph-BR の価値が上がる)。
- 留保: lock_active スライス0件(ビット解釈要確認・ロック未検証)。A0は最穏スライス(序盤層+ストックでは差が
  拡大しうる)。タイムが真に弱い可能性も残る(が、それを証拠付きで言うにはレジーム厳密値が要る)。
- 反映: 挟み撃ちテーブルはタイム/ストック次元込みで作る・n7_regime_slices を常設計器化・レジーム内厳密値を
  錨に明示的に効かせる(中立)・graph-BR。

### 6-4. 推奨ロードマップ
① C の njit 化(全ての鍵)→ ② graph-BR 計器 → ③ (1,1)厳密テーブル接続(層挟み撃ち)→ ④ 深い教師で再学習→②で検証、反復。
