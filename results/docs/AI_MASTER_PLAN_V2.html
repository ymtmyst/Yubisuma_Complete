Complete AI マスタープラン V2 — ミラー・リバーシ OFF 最強 AI

作成日: 2026-07-12 作成方法: 旧計画(complete_ai_plan.html / AI_PROGRESS_CHECKLIST.md)を前提にせず、コード・ルール・状態空間をゼロベースで再調査して策定 対象設定: RulesConfig(enable_mirror=False, enable_reversi=False) に固定(ミラー・リバーシ OFF) 運用: AI_WORKFLOW_RULES.md に完全準拠(各フェーズ末に確認成果物を提示し、ユーザー承認まで次工程に進まない)

1. 目的と「最強」の定義

目的はミラー・リバーシ OFF の Complete ルールで最強の AI を作ること。

このゲームは 完全情報・決定論・同時手番・二人ゼロ和のマルコフゲーム である(complete_solver/transition.py で確認済み。乱数なし、隠れ情報なし、同時宣言のみ)。この構造のゲームには一意のゲーム値と定常マルコフ完全均衡(マキシミン戦略)が存在し、各状態の最適混合戦略は「子状態の値を並べた行列ゲームを LP で解く」ことで厳密に求まる。

したがって「最強」を次の3条件で定義する。

  1. 非搾取性(近似ナッシュ): どんな相手戦略にも期待値で大きく負けない。混合戦略を含む均衡プレイ。
  2. 搾取性: 弱い相手(none/random 等の固定方策、旧モデル)には最短手数で確実に勝つ。
  3. 人間妥当性: 行動分布・初手・スキル使用が人間視点で説明可能(AI_WORKFLOW_RULES の目視ゲート)。

測定方法は §6 の採用ゲートで固定する。勝率単独では判定しない。

2. ゼロベース調査で確立した事実(2026-07-12 計測)

計画の根拠となる一次データ。すべて本日実測・実読した。

  1. ルールエンジンは高品質で再利用可能。 complete_solver/(state / actions / transition / matrix_game / finite_horizon / state_space)は純粋関数・immutable 設計で、132 テストが PASS。視点交代の符号処理(same_turn_player)も遷移層で正しく扱われている。これはプロジェクトの中核資産であり全面的に残す。

  2. 全状態列挙(=全域厳密解)は不可能。 ミラー・リバーシ OFF で初期状態から BFS 列挙を実測した結果、8 分で到達 54.1 万状態・未展開フロンティア 43.3 万・拡張 1 回あたり新規状態 4〜7 個のまま減速せず。ストックの部分集合(2^8)× セメント × バフ類 × 両者分の組合せで、到達可能空間は少なくとも 10^7 オーダー以上と推定される。「小さいゲームだから値反復で終わり」は成立しない。 旧計画の max_states=400〜500 の値反復は全体のごく一部を leaf=0 で打ち切った近似であり、そこから作った BC 教師(zero leaf)が数字偏重の根本原因だったことと整合する。

  3. 深さ制限 LP 探索は既に存在するが遅い。 FiniteHorizonSolver は「LP バックアップ付き深さ制限探索」そのもの(=本計画の実戦エンジンの原型)。ただし素の Python で初期局面(TP 45 手 × NTP 9 手)の depth=3 が 15.7 秒、depth=4 が 121 秒。高速化が第一級の課題。

  4. GPU 環境あり。 Python 3.10 / torch 2.6.0+cu124 / CUDA 利用可。価値ネット学習は問題なし。

  5. 旧路線(BC + MaskablePPO)の失敗は偶発ではなく構造的。 - zero-leaf VI 教師 → 数字 100% 偏重(境界アーティファクト) - 反応履歴 16 次元の観測 → 状態が非マルコフ化し、均衡概念と評価指標(exploitability)が壊れた - 決定論的初手(フェイント固定・スキップ固定)→ 同時手番ゲームでは決定的方策は原理的に搾取可能。ent_coef 調整では解決しなかった(item 42) - none 相手の無限ループ打ち切り → モデルフリー RL が「勝ち筋の発見」を報酬信号だけで行う限界 これらは PPO の調整不足ではなく、モデルフリー・方策ネット中心のアプローチがこのゲームの既知構造(完全情報・厳密遷移・LP で解ける局所均衡)を捨てていることに起因する。

3. 新方針: 探索中心アーキテクチャ(LP バックアップ探索 + 価値ネット + 終盤厳密テーブル)

AlphaZero の構図を同時手番ゼロ和ゲーム用に置き換えたもの。方策ネットを主役にせず、探索を主役にする。

実戦時(プレイ)

着手 = 深さ制限探索(depth 2〜4, 適応)
        ├─ 各ノード: 子の値を並べた行列ゲームを LP で解き、値と混合戦略を得る
        ├─ 葉: 学習済み価値ネット V(s)(107 次元特徴量)
        ├─ 終盤・低複雑度状態: 厳密テーブルの値を直接参照
        └─ ルートの LP 混合戦略から実際に「サンプリング」して着手(決定論にしない)
  • 混合戦略サンプリングが仕様の中心。 同時手番ゲームの均衡は本質的に混合であり、これが旧モデルの「初手固定を突かれる」問題を原理レベルで消す。
  • none のような受動的相手には探索が数手先の強制勝ちを直接発見する。γ<1 により最短勝利が選好されるため、無限ループ打ち切りは起こり得ない。

学習時(fitted Nash value iteration)

繰り返し:
  1. 自己対戦で状態を収集(双方が現行探索方策、LP 混合からサンプル + ε ランダム注入)
  2. 収集した各状態に「depth-d LP バックアップ値(葉=現行ネット)」を教師値として付与
  3. 終端値・終盤厳密テーブル値をアンカーとして混ぜ、V(s) を回帰学習
  4. 新旧ネットで世代対戦・テーブル一致度を測り、改善していれば採用

これはマルコフゲームに対する fitted Nash-VI(minimax-Q の関数近似版)であり、完全情報ゼロ和では値が一意なので、方策の非定常性・忘却(PPO で起きた catastrophic forgetting)の問題構造がそもそも生じない。学習対象は「値」だけで、方策は常に探索+LP が現場で導出する。

旧路線の問題がどう消えるか(対応表)

旧問題 本アーキテクチャでの帰結
zero-leaf 教師の数字偏重 葉は学習済み V + 終端/テーブルアンカー。leaf=0 境界が存在しない
初手決定論(フェイント/スキップ固定) LP 混合戦略からサンプリング。均衡混合が仕様として保証
none 相手の無限ループ 探索が強制勝ちを直接読む。γ<1 で最短勝利選好
反応履歴で非マルコフ化 状態は完全観測でマルコフ。履歴特徴量は不要(廃止)
exploitability 指標の破綻 BR 攻撃エージェントによる実測搾取率に置換(§6)
PPO の視点符号バグ類 solver の遷移層(検証済みコード)だけを使う

5. フェーズ計画

各フェーズの末尾に確認成果物(HTML/表)+ AI の解釈を提示し、ユーザー承認後に次へ進む。

N1. 高速探索基盤(最優先・計画全体の律速)

  • 内容:
  • 遷移・合法手生成の結果キャッシュ、状態の安定ハッシュ、探索間で持続するトランスポジションテーブル
  • 純粋鞍点チェック(行列に純粋均衡があれば LP を呼ばない — 大半のノードで LP 回避見込み)
  • ノード内ダブルオラクル(45×9 の全子展開をやめ、ベストレスポンスを交互に追加して小さい部分行列で均衡を得る)
  • 明白劣位行動の枝刈り(例: 合計 0〜4 の数字宣言のうち文脈上無意味なもの)
  • 完了条件: 既存 132 テスト PASS のまま、初期局面 depth=3 ≤ 1 秒・depth=4 ≤ 10 秒(現状 15.7s / 121s)。
  • 確認成果物: 高速化前後の値・方策の一致検証表、速度ベンチ HTML。
  • 判断点: 目標未達なら Numba / Rust コア化をここで見積もる(先送りしない)。

N2. 終盤・低複雑度厳密テーブル

  • 内容: 「ストック空 or 小」「hands 合計が少ない」等の部分空間を実測列挙し、規模が許す範囲(目安 10^5〜10^6 状態)を厳密値反復で解いてディスク保存。探索から O(1) プローブ。
  • 完了条件: テーブル規模・値分布・代表局面の妥当性レポート。探索接続テスト PASS。
  • 意義: (a) 実戦で終盤は文字通り最適、(b) 価値ネットの正解アンカー、(c) ネット精度の恒久的な検証基準。

N3. 価値ネット v0

  • 内容: 107 次元 → MLP(目安 2〜3 層 × 512)。初期教師データ: ランダム/浅い探索プレイアウトで収集した状態への depth-2〜3 バックアップ値 + N2 テーブル値。
  • 完了条件: N2 テーブル上での MSE・順位相関レポートで material_leaf_evaluator を明確に上回る。
  • 確認成果物: 代表局面の V(s) と厳密値の比較表。

N4. fitted Nash-VI 自己対戦ループ(本体)

  • 内容: §3 の学習ループを世代制で回す。世代ごとに (a) 新旧ネット世代対戦、(b) テーブル一致度、(c) 教師値の分散・発散チェック。ε ランダム状態注入で分布カバレッジを確保。
  • 完了条件: 3 世代以上連続で世代対戦・テーブル一致度が悪化しないこと。行動分布 HTML の人間確認。
  • リスク対応: 発散したらターゲットネット固定・学習率調整・アンカー比率増で戻す(その時点で停止して報告)。

N5. 実戦エージェント確定

  • 内容: 時間予算に応じた適応深度、テーブルプローブ、混合サンプリング(検証用に決定論モードも用意)、人間対戦用 CLI。
  • 完了条件: 対戦 CLI で人間がプレイ可能。挙動レポート承認。

N6. 評価ゲート(§6 を実施)

  • 内容: named NTP 方策一式・旧ベスト PPO 2 モデルとの対戦、BR 攻撃(既存 MaskablePPO 基盤を攻撃者として凍結エージェントに 250k steps × 複数 seed 訓練)、行動レポート。
  • 完了条件: §6 の全ゲート通過 + ユーザー最終承認。

N7. 反復強化(継続) — 「深さを掘る以外」で強くする

戦略考察と設計案の全文は N7_STRATEGY_AND_DESIGN.md。要旨:

  • 族(LP バックアップ探索 + 価値ネット + fitted Nash-VI)は変えない。 これは AlphaZero を同時手番 ゼロ和に正しく一般化した先。MuZero/CFR/PSRO への乗り換えは真のボトルネック(長期スキルの信用割当)を 解かず、完全情報+厳密シミュレータの強みを捨てるので不採用。
  • 深さ4は主戦略にしない。 遅効性スキル(セメント/ストック/ロック)は約10プライ先で報われ、深さ4 (葉+2プライ)では構造的に届かない。真のレバーは 価値ネットの長期精度と訓練カバレッジ (ネットが完璧なら深さ1で足りる=深さ探索は穴埋めにすぎない)。

採用する強化路線(優先度順):

  • A(本命)— 到達可能サブグラフ Nash 値反復: 決定論+完全情報を使い切り、深さを掘らずに長期価値を サブグラフ全域へ伝播させて教師値を作る。A0 厳密DBの機構(packed_vi の _enumerate/_jacobi_sweep/ PackedEndgameDB、A0=33.4万状態を54スイープ25秒で収束の実績)を「閉じた宇宙→サンプリング領域+ フロンティアをネットで閉じる」形に一般化。教師差し替えで B/C/D すべての土台になる。
  • B — 価値ネットの構成エンコーダ化: フラット特徴で潰れる非線形スキルコンボを、保持スキル集合/盤面への 集合・注意エンコーダで表現。A の後(厚い教師の後)が相性最良。
  • C — 選択的/最良優先探索(smarter, not deeper): 均衡台(support)上の重要ラインだけ深掘りする反復 深化で、深さ4同コストで実効深さ6〜8。A と並行可。
  • D — 深さ4教師: インフラ完成済みで安価。A の効果測定の対照(=深さでは届かないことの実証)に随時活用。

(旧 N7 メモ: depth 引き上げ、データ増、ネット拡大、コアの Rust 化、γ 感度 0.997〜0.999 最終確定、 弱点局面の的当て学習 — これらは上記に統合/継続。)

想定順序: A(先行)→ C は A と並行可 → B は A の後 → D は随時。各フェーズ末に確認成果物+解釈を提示し ユーザー承認後に次へ。

想定順序は N1 → N2 → N3 → N4 → N5 → N6(→ N7 反復)。N2 と N3 は一部並行可。

6. 採用ゲート(最終合格条件)

  1. 固定方策制圧: none / counter_first / block_first / random / 分離系(none_uniform 等)すべてに勝率 ≥ 99%、打ち切り 0(none 含む)。
  2. 旧世代超え: 旧ベスト PPO(maskable_ppo_bc_standard_episode_mixed、maskable_ppo_bc_depth4_standard_proper)との直接対戦で勝率 ≥ 70%。
  3. 実測非搾取性: 凍結した本エージェントに対し BR-PPO 攻撃者(250k steps × 3 seeds)を訓練しても、本エージェント側勝率 ≥ 55% を維持。
  4. 人間妥当性: 初手分布・スキル使用率・開始側手順レポートで警告 0 + ユーザー目視承認。
  5. 上記すべてを 開始側/後手側の両視点で集計する(旧レポートの視点混同バグの教訓)。
