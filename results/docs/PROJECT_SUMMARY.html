Yubisuma Complete AI — 全体像まとめ（2026-07-13）

一言でいうと

7週間止まっていた「BC+強化学習」路線を一度捨て、ゼロから探索中心の設計で作り直し、途中で発覚した根本的なルール実装バグも直しきって、「ミラー・リバーシOFF版の最強AI v1」に相当する状態まで到達しました。

1. スタート地点（あなたが戻ってきた時点）

前任の下位モデルが作った旧AIは、勝率は出るものの「数字を92〜96%連打する」「none相手に無限ループで打ち切る」といった不自然な病理を抱え、採用保留のまま止まっていました。あなたの「手段は問わない、ゼロベースで考え直してほしい」という依頼から始まりました。

2. 方針の再設計（N0）

ゲームの性質を調べ直しました — 完全情報・決定論・同時手番・ゼロ和。この構造では各局面の最適な「混ぜ方（混合戦略）」はLP（線形計画）で厳密に計算できます。旧路線の失敗は「AIに混ぜ方まで学習させようとした」ことにあると突き止め、混ぜ方は毎手LPで厳密計算し、ニューラルネットには局面の点数だけを学ばせる将棋AI（AlphaZero）型の設計に切り替えました。これが初手固定・無限ループといった病理を原理から消す狙いです。

3. 段階的に構築（N1〜N6）

段階 やったこと 平易に言うと
N1 深さ制限LP探索を約8倍高速化 AIの「読み」を速くした
N2 ルールエンジンをNumbaで約1000倍高速化、終盤厳密テーブル 「詰みデータベース」の基盤+超高速化
N3 103次元特徴の価値ネット学習 AIの「大局観」の初代を作り、手作り評価に圧勝
N4 自己対戦の世代反復 AI同士を戦わせて大局観を鍛える工場
N5 実戦エージェント+対戦CLI+評価台 あなたが遊べる形+実力測定
N6 BR攻撃テスト 弱点を突く専属攻撃者に耐えるか検証

4. 途中の大事件：ルール実装バグの発覚

あなたが方策分布レポートを見て「コピー（フェイント）が反応なしで決着76.8%はありえない」と指摘したのが決定打でした。調べると、5月からソルバーと対話版の両方が同じ誤解釈を共有していて（だから相互照合では見つからなかった）、「フェイントは相手がカウンターした時だけ発動する」というルールを無視して無条件発動させていました。

これで全成果物が無効に。そこから、あなたの裁定を仰ぎながら次々と修正しました： - 参照経由のフェイント/ロックを正しく（カウンター時のみ発動） - ガードを「効果全体が1フェーズ1回」に（ルール文書の改訂に対応） - タイム、スキップ+タイムの手番奪取 - 「パスというスキルは存在しない」というあなたの指摘で、疑似パス手番を廃止し真のスキップに

さらに再構築中に価値反復が発散する数値バグ（3連鎖）も特定・解決しました。ルールを直すたびに全成果物を自動で再生成する仕組みを作ったので、都度きれいに作り直せています。

5. 今のAIの実力（再定義した合格基準を全クリア）

基準 結果
① 手作り評価より強い ✅ 61〜66%で勝ち越し
② 病理（無限ループ）なし ✅ 混合戦略で打ち切りほぼゼロ
③ 旧最強モデル2種に勝ち越し ✅ 旧best1に63〜70%、旧best2に53〜57%
④ 搾取されない ✅ 専属攻撃者3体すべて勝率48%未満（パリティ超え不可）

固定相手には68〜77%で勝ち、方策も「初手フェイント/終盤に数字」といった人間的で健全な分布になりました（旧病理は消滅）。

6. 残っている課題（あなたの診断が的中している部分）

セメント・ストック・ロック・タイムといった長期スキルは、まだほとんど使えていません。 これはあなたの「深さが浅いと効果が10手先に出る長期スキルの価値が見えない」という信用割当の診断どおりで、証拠に「終盤セメント>序盤セメント」という逆転が残っています。ただし対策（教師の読みを深くする）は効くことが実証済みで、深さ3にしたらロック・ストックが終盤から芽吹き始めました。「深さを買うほど良くなる」構造が確認できたので、次はここへの投資判断になります。

7. 副産物：整理とレポートハブ

  • ディスクを 8.2GB → 198MB に（旧RL遺物8GBを削除、Basic版の成功解析は無傷で保護）
  • results/をきれいな階層に再編（現行7枚+旧物は_legacy/に隔離）
  • Complete/results/index.html — 1枚開けば全レポートに飛べる集約ハブを作成。折りたたみセクション・状態バッジ（🔥進行中/🧱基盤/📚参考）・更新日つき。マークダウンや完全ルール（新）はresults/docs/に整形HTMLとして自動生成され、生テキストではなく読みやすい形で開ける。

📌 今後の運用ルール：新しいレポート／ドキュメントを追加したらハブに登録する

新しい成果物（レポートHTML・計画/ログ等の.md）を作ったら、必ずレポートハブに追記すること。 手順は登録元1ファイルを編集して再生成するだけ：

  1. Complete/complete_ai/build_report_hub.py を開く。
  2. 追加物の種類に応じてリストに1行足す： - フェーズ別レポート（results/内のHTML）→ CURRENT に (タイトル, ファイル名, 説明, バッジ) を追加 - 計画・記録・ルール系の.md／独自記法ドキュメント → DOCS に (タイトル, 元ファイル, "md" または "rule", 出力名.html, 説明, バッジ) を追加（docs/に整形HTMLが自動生成される） - 終盤シナリオ → ENDGAME、旧診断は_legacy/diagnostics_may/に置けば自動収集される
  3. バッジは現状に合わせて選ぶ："hot"=いま動かしている最新／"base"=土台・安定／"ref"=参考・アーカイブ寄り。既存物の重要度が変わったらここの"hot"/"base"/"ref"を書き換える（更新日は各ファイルの更新時刻から自動表示なので手入力不要）。
  4. python -m complete_ai.build_report_hub を実行してindex.htmlとdocs/を再生成。

※ index.htmlを直接編集しても再生成で上書きされる。編集は必ずbuild_report_hub.py側で行うこと。
