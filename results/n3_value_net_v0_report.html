N3 価値ネット v0 — 学習と精度検証

作成: 2026-07-12 / 対象: ミラー・リバーシOFF / gamma=0.999 / GPU (CUDA)

1. 構成

  • 特徴量(complete_ai/features.py、103次元): 両プレイヤーの手・セメント・バフ・ストック one-hot に加え、ユーザードメイン知識を反映した明示特徴(ストック保持数、対カウンター系保持フラグ、通常系保持フラグ)、前手 one-hot、追加ターン。Numbaバッチ版とPython参照版が同一コード経路。
  • データ: コンパイル済みエンジンでスキル偏重ランダムプレイアウト(開幕根6万局+(1,1)終盤根4万局)→ 116万訪問 → ユニーク13.5万局面。
  • 教師値: 各局面の「深さ2 LPバックアップ値」(葉=material評価)。正しさは FastHorizonSolver との一致テストで担保。
  • ネット: MLP 103→512×3→1 (tanh)。学習3秒/60エポック(早期終了付き)。models/value_v0.pt

3. AIの解釈

  • v0の目的は「パイプラインが正しく動き、学習が手作り評価を明確に超えること」の確認であり、3つのものさし全てで大差クリア。
  • 特にA0厳密テーブル(学習教師とは独立の真値系)で順位相関 0.34→0.92 に跳ねたことは、ネットが単なる暗記でなく局面の質を学んでいる証拠。
  • ただし v0 の限界も明確: (a) 教師が深さ2+material葉なので値の絶対水準はまだ浅い読みの水準、(b) 学習分布がランダムプレイアウト由来で強者同士の実戦分布と違う、(c) 13.5万局面は小さい。これらは N4(自己対戦の fitted Nash-VI 反復)がまさに解決する項目。

4. 実装上の事件記録(再発防止)

  • 行動数バッファ(64行)を満杯ストック局面のTP行動数(最大78)が超え、ネイティブクラッシュ(exit 116)が2回発生。バッファを96行に拡大し、>64行動局面の回帰テストを追加して解決。
  • 教訓として「モデルは学習直後に保存」「評価は別プロセスで段階ログ付き」に構造化。
