1. ベースラインの定義(旧評価との違い)

旧RL環境の named policy 評価は「テスト対象モデルが相手側のTP宣言まで打つ」曖昧仕様だった。 今回は両席を完全分離: スクリプト相手はTP宣言=一様ランダム、NTP反応だけスタイル固定 (random=一様 / none=常に反応なし / counter=可能なら常にカウンター / block=ブロック優先)。 先手後手は交互、300局ずつ。

3. 発見(重要)

  1. LP混合サンプリングが「none無限ループ」を完治させている実証: 決定論モードは反応なし相手に177/300が打ち切り(旧PPOと同じ病理が決定論プレイで再現)。 混合モードは同条件で打ち切りゼロ。「均衡混合から着手する」という本計画の中核設計の正しさが、 理論でなく対局データで確認された。
  2. ネットの寄与はどの相手でも一貫(特に vs none: 55%→89%)。
  3. 勝率の天井問題: このゲームは「自分の数字を当てて自分の手を減らすレース」であり、 ランダムに宣言する相手でも運で先に上がる確率が内在する。旧計画の「固定方策に勝率99%」ゲートは、 相手が宣言しない旧仕様の産物であり、両席分離の公正な評価では達成不可能な見込み。 material探索・決定論・混合のいずれも75〜90%帯に収まることがこの解釈を支持する。

4. 対戦CLI

python -m complete_ai.play_cli(AI先手は --ai-first)。 同時手番性を保持: あなたが宣言する時はAIは反応を先にコミット、AIが宣言する時はあなたの反応を先に入力。 自動スモークで対局完走を確認済み。

4.5 N5b: 旧PPOベストとの直接対戦(200局×反応3種、先手交代)

旧アーキテクチャはTP宣言しか学習していない(反応は環境の固定方策任せ)ため、旧側の席は 「旧モデルの宣言+スクリプト反応(random/counter/block の3種を試し、旧側に最も有利な値も併記)」 という旧側に寛大な設計で対戦した。

5. 提案(ユーザー判断事項)

  • 採用ゲートの再定義(旧「99%」は不成立): ①全ベースラインに対して material 対照より明確に優位、 ②打ち切り病理なし(混合モード)、③旧PPOベスト2種に直接対戦で勝ち越し、④BR攻撃に対する非搾取性(N6)、 ⑤ベストレスポンス推定値(相手固定時の理論上限)との差が小さいこと — を新ゲートとする。
