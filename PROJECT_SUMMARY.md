## 4. 途中の大事件：ルール実装バグの発覚

**あなたが方策分布レポートを見て「コピー（フェイント）が反応なしで決着76.8%はありえない」と指摘したのが決定打**でした。調べると、5月から**ソルバーと対話版の両方が同じ誤解釈**を共有していて（だから相互照合では見つからなかった）、「フェイントは相手がカウンターした時だけ発動する」というルールを無視して無条件発動させていました。

これで全成果物が無効に。そこから、あなたの裁定を仰ぎながら次々と修正しました：
- 参照経由のフェイント/ロックを正しく（カウンター時のみ発動）
- ガードを「効果全体が1フェーズ1回」に（ルール文書の改訂に対応）
- タイム、スキップ+タイムの手番奪取
- **「パスというスキルは存在しない」**というあなたの指摘で、疑似パス手番を廃止し真のスキップに

さらに再構築中に価値反復が発散する数値バグ（3連鎖）も特定・解決しました。ルールを直すたびに全成果物を自動で再生成する仕組みを作ったので、都度きれいに作り直せています。

## 6. 残っている課題（あなたの診断が的中している部分）

**セメント・ストック・ロック・タイムといった長期スキルは、まだほとんど使えていません。** これはあなたの「深さが浅いと効果が10手先に出る長期スキルの価値が見えない」という信用割当の診断どおりで、証拠に「終盤セメント>序盤セメント」という逆転が残っています。ただし対策（教師の読みを深くする）は効くことが実証済みで、深さ3にしたらロック・ストックが終盤から芽吹き始めました。**「深さを買うほど良くなる」構造**が確認できたので、次はここへの投資判断になります。

### 📌 今後の運用ルール：新しいレポート／ドキュメントを追加したらハブに登録する

**新しい成果物（レポートHTML・計画/ログ等の.md）を作ったら、必ずレポートハブに追記すること。** 手順は登録元1ファイルを編集して再生成するだけ：

1. **`Complete/complete_ai/build_report_hub.py`** を開く。
2. 追加物の種類に応じてリストに1行足す：
   - フェーズ別レポート（`results/`内のHTML）→ `CURRENT` に `(タイトル, ファイル名, 説明, バッジ)` を追加
   - 計画・記録・ルール系の`.md`／独自記法ドキュメント → `DOCS` に `(タイトル, 元ファイル, "md" または "rule", 出力名.html, 説明, バッジ)` を追加（`docs/`に整形HTMLが自動生成される）
   - 終盤シナリオ → `ENDGAME`、旧診断は`_legacy/diagnostics_may/`に置けば自動収集される
3. **バッジ**は現状に合わせて選ぶ：`"hot"`=いま動かしている最新／`"base"`=土台・安定／`"ref"`=参考・アーカイブ寄り。既存物の重要度が変わったらここの`"hot"/"base"/"ref"`を書き換える（更新日は各ファイルの更新時刻から自動表示なので手入力不要）。
4. **`python -m complete_ai.build_report_hub`** を実行して`index.html`と`docs/`を再生成。

※ `index.html`を直接編集しても再生成で上書きされる。編集は必ず`build_report_hub.py`側で行うこと。
