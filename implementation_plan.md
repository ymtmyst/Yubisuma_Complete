# 指スマ完全ルール版 AI — 強化学習手法比較分析

## 1. ゲーム固有の性質分析

手法を比較する前に、このゲームの性質を精密に把握する。**ゲーム一般ではなく、この指スマ完全ルールの特性**に最適化された手法を選ぶことが最も重要である。

### 1.1 ゲーム構造

| 特性 | 詳細 |
|:---|:---|
| **プレイヤー数** | 2人（二人零和ゲーム） |
| **情報構造** | **不完全情報**（同時意思決定）— 相手の指の本数・カウンター/ブロック選択は同時公開まで不明 |
| **ターン構造** | 交互ターン制（ただし追加ターン・タイム効果で変動） |
| **終了条件** | どちらかの手が0になったら勝利 |
| **確定性** | 確定的（乱数要素なし、ゲームルール自体に運要素はない） |

### 1.2 状態空間分析

```
各プレイヤーの状態:
  - 手の数: 3通り (0, 1, 2)
  - ガード: 2通り (ON/OFF)
  - チャージ: 2通り
  - クイックレベル: 3通り (0, 1, 2)
  - セメント: 4通り (None, 0, 1, 2)
  - ロックデバフ: 3通り (0, 1, 2)
  - スキップフェーズ: ~3通り (0, 1, 2+)
  - 必殺使用済み: 2通り
  - タイム: 2通り
  - ストック: 可変長リスト(8種のスキルの部分集合)
  - チョイス使用済み: フェーズ内ローカル
  - ドロップ封印: スキル集合

グローバル状態:
  - ターン履歴 (前ターンのスキル)
  - 先手制限フラグ
  - 追加ターン数
  - スキップ連鎖フラグ
```

> [!IMPORTANT]
> **状態空間の推定サイズ**: 各プレイヤーの基本バフ/デバフ状態だけで約 `3×2×2×3×4×3×3×2×2 = 5,184通り`。2人分で約 `5,184² ≈ 2,700万`。ストック・ターン履歴を含めると数億〜数十億オーダーに達するが、**到達可能な状態はごく一部**であり、ゲーム木の実効的な深さも限定的（通常10〜30ターン程度）。

### 1.3 行動空間分析

```
ターンプレイヤー (TP) の行動:
  - 数字の宣言: 0〜4 の整数 (5通り)
  - スキル宣言: 最大15種のスキル（条件により制限）
  - 指の本数: 0〜2 (3通り、セメントにより制限)
  → 組み合わせ: 最大 (5+15) × 3 = 60通り

非ターンプレイヤー (NTP) の行動:
  - リアクション: カウンター / ブロック / なし (3通り、条件により制限)
  - 指の本数: 0〜2 (3通り)
  → 組み合わせ: 最大 3 × 3 = 9通り
```

> [!NOTE]
> **行動空間は離散的で比較的小さい**（最大60 × 9 = 540の同時行動組み合わせ）。これはDQN系手法にとって有利な特性。

### 1.4 このゲームの核心的特徴

| 特徴 | RL設計への影響 |
|:---|:---|
| **同時意思決定** | 相手の行動が見えない状態で自分の行動を決定。Nash均衡的な混合戦略が最適解 |
| **スキル間の複雑な相互作用** | チャージ→数字、クイック→クイック、コピー→前ターン参照など、状態遷移が非線形 |
| **一発上がりの存在** | フラッシュ・チャージなどで一撃勝利可能 → 局面評価に大きな非連続性 |
| **ストック/チョイスの組合せ爆発** | 蓄積したスキルの選択肢により、ゲーム後半で分岐が爆発的に増加 |
| **対カウンター読み合い** | フェイント/ロックはカウンターされた時のみ発動 → 相手の行動予測が重要 |
| **先手制限ルール** | 非対称な開始条件 → 先手/後手を別エピソードとして学習する必要あり |

## 3. 各手法の詳細分析

### 3.1 Tabular Q学習

- **状態空間が広すぎて不可能** — あなたの分析の通り。ストック・ターン履歴を含めると状態数は数億規模であり、テーブルベースは現実的でない。
- **結論: ❌ 不採用**

### 3.2 Vanilla DQN

- 離散行動空間と相性は良いが、**不完全情報の同時意思決定に対する対策がない**
- 自己対戦で学習させると非定常環境問題（相手の方策が変化）でQ値が不安定化
- **結論: ❌ 単体では不十分**

### 3.3 DQN発展版 (Double + Dueling + PER)

- Double DQN: 過大評価バイアスの軽減
- Dueling DQN: 状態価値とアドバンテージの分離 — **多数のスキルの中で「どのスキルが有利か」の学習が効率化**
- PER (Prioritized Experience Replay): 一発上がりなどの稀なイベントを重点学習
- 不完全情報対応は依然として弱い
- **結論: △ DQN系を使うなら最低限この構成**

### 3.4 Rainbow DQN

- 6つの改良を統合するため、ハイパーパラメータ調整が困難
- このゲームの状態空間サイズには**オーバースペック**
- Distributional RL (C51) やNoisy Netsの恩恵がこのゲームでは限定的
- **結論: ▲ 労力に見合うリターンが薄い**

### 3.5 ⭐ PPO (Self-Play) — **最推奨**

- **安定性が最大の強み**: クリッピング機構により方策更新が安定
- **Self-Play**: 相手も自分と同じ方策を使って学習 → 方策の共進化
- **不完全情報への対処**: 観測に含まれる情報のみから行動を決定する構造が自然に「情報集合ベースの意思決定」を実現
- **混合戦略の自然な学習**: 方策ネットワークが確率的方策を直接出力するため、Nash均衡的な混合戦略を自然に学習可能（**DQNは決定的方策のためこれが困難**）
- Stable Baselines3等の成熟したライブラリが使用可能
- **2024-2025年の研究で、PPOが不完全情報ゲームでCFRと同等以上の成績を達成することが示されている**

> [!TIP]
> **PPOが混合戦略を学習できることが決定的に重要**。指スマの同時意思決定（指の本数選択・カウンター判断）ではNash均衡が混合戦略になる場面が多い。DQN系は決定的な行動選択であり、ε-greedyによる擬似的なランダム性では最適な混合比率を学習できない。

- **結論: ✅ 最も推奨。実現可能性・安定性・理論的妥当性のバランスが最良**

### 3.6 PPO + 行動予測ヘッド（発展版）

- PPOのネットワークに**相手の行動予測ヘッド**を追加する拡張
- 補助損失として「相手がカウンターを使うか」「相手の指の本数」を予測するタスクを追加
- 不完全情報の効果的な活用が可能に — **このゲームのカウンター読み合いに特に有効**
- 実装はPPOベースに補助ヘッドを追加するだけで済む
- **結論: ✅ 時間に余裕があれば次のステップとして推奨**

### 3.7 CFR / Deep CFR

- **理論的にはNash均衡への収束が保証される** — 不完全情報ゲームの「正解」アルゴリズム
- しかし、このゲームの情報集合の列挙が複雑（ストック・スキル連鎖の組み合わせ）
- Deep CFRはニューラルネットで情報集合を近似するが、実装は非常に複雑
- ライブラリサポートが限定的（OpenSpielにはあるが、カスタムゲーム統合は手間）
- **結論: △ 理論的には最強だが実装コストが高すぎる**

### 3.8 NFSP (Neural Fictitious Self-Play)

- DQNの最良応答学習 + 平均方策ネットワークの組み合わせ
- Nash均衡への近似収束が期待できる
- DQNベースのため混合戦略の表現力に課題あり
- 2つのネットワーク（最良応答+平均方策）の同時訓練が不安定になりがち
- **結論: △ 理論は面白いが安定性で劣る**

### 3.9 R-NaD / DeepNash式

- **正則化Nash動力学**: 強い理論的保証でNash均衡に収束
- DeepMindがStrategoで超人的性能を達成
- しかし、**論文の再現が極めて困難**（公開実装が限定的）
- 大規模な計算リソースが必要
- **結論: ▲ 学術的には最先端だが個人実装は非現実的**

### 3.10 AlphaZero / MuZero (MCTS + RL)

- あなたの直感は正しい — **標準形では不完全情報ゲームに非対応**
- 「カウンターの有無」「相手の指の本数」が観測できない状態でMCTSのシミュレーションができない
- IS-MCTS (Information Set MCTS) への拡張は理論的に可能だが、実装が極めて複雑
- **あなたの指摘する「離散空間で分岐を総当たりできる」点は正しいが、「相手の隠された選択（カウンター/指の本数）」の期待値計算が必要であり、純粋なMCTSでは不十分**

> [!WARNING]
> AlphaZero/MuZeroは**完全情報ゲーム用**のアーキテクチャ。このゲームの同時意思決定（TPの指の数とNTPのカウンター判断が互いに見えない）に対して、標準MCTSは理論的に最適でない解を出す可能性がある。

- **結論: ❌ 不採用（修正版でも実装コストが膨大）**

### 3.11 ReBeL (PBS + Search + RL)

- Public Belief State (PBS) という概念で不完全情報を理論的に正しく扱う
- Nash均衡への収束保証あり
- しかし**実装が極めて複雑**で、ポーカー以外への適用例が限定的
- **結論: ❌ このゲーム規模には過剰（オーバーエンジニアリング）**

## 4. このゲーム固有の判断ポイント

### なぜPPOが最適か — ゲーム性質との対応

| ゲームの性質 | PPOの対応 | DQN系の対応 |
|:---|:---|:---|
| 同時意思決定（混合戦略が必要） | ✅ 確率的方策を直接出力 | ❌ 決定的方策、混合戦略を学習不可 |
| カウンター読み合い | ✅ 確率的にカウンター/なしを選択する方策を学習 | ▲ ε-greedyでは最適な確率を学習不可 |
| 一発上がり（報酬の非連続性） | ✅ GAE (Generalized Advantage Estimation) で対処可 | △ PERで一部対処可だが根本的でない |
| スキル間の複雑な遷移 | ✅ 方策ネットワークが状態→行動確率の写像を学習 | △ Q値推定が複雑な遷移で不安定化しやすい |
| 学習の安定性 | ✅ クリッピングで安定 | ▲ 自己対戦環境で不安定化しやすい |
| ライブラリサポート | ✅ SB3, Ray RLLib等 | ○ SB3等 |

### 行動空間サイズの妥当性

- **最大60行動**: PPOにとって問題ない規模
- **動的マスキング**: 状態に応じて使用不可能なスキルをマスクする機構が重要
  - 例: スキップ中は全スキル不可、先手1フェーズ目はフラッシュ/必殺不可
  - SB3の`MaskablePPO`が直接対応可能

## 5. 推奨実装ロードマップ

### Phase 1: PPO Self-Play（基盤構築）

```
1. ゲーム環境をGymnasium互換に実装
   - observation_space: 全状態情報をフラットなベクトルに変換
   - action_space: スキル選択 + 指の本数を1次元離散空間にエンコード
   - 無効な行動のマスキング（MaskablePPO対応）

2. Self-Play訓練ループ
   - 最新の方策 vs 過去の方策チェックポイントで対戦
   - 先手/後手を交互に担当

3. 報酬設計
   - 勝利: +1, 敗北: -1
   - 中間報酬: 手を降ろした時のボーナス（任意）
```

### Phase 2: 拡張（行動予測ヘッド追加）

```
4. 補助タスクの追加
   - 相手の行動予測ヘッド（カウンター確率、指の本数分布）
   - マルチタスク学習で不完全情報の活用を改善

5. League Training
   - 複数世代の方策と対戦するリーグ方式
   - 特定の戦略への過適合を防止
```

### Phase 3: 評価・改善

```
6. ルールベースAIとの対戦評価
7. 戦略分析（どのスキルをどの状況で使うか可視化）
8. 必要に応じてネットワーク構造・報酬関数の改善
```

## 6. 最終結論

> [!IMPORTANT]
> ### 推奨手法: **PPO (Self-Play) + MaskablePPO + 動的行動マスキング**
> 
> **理由:**
> 1. **混合戦略の自然な学習** — 同時意思決定ゲームに不可欠
> 2. **学習安定性** — Self-Play環境でも安定した学習が可能
> 3. **実装の現実性** — SB3/MaskablePPOによる即戦力ライブラリ
> 4. **拡張性** — 行動予測ヘッド、League Training等への段階的拡張が容易
> 5. **最新研究の裏付け** — 2024-2025年の研究でPPOが不完全情報ゲームでCFRと同等以上の成績
>
> DQN系は混合戦略の学習が本質的に困難であるため、**このゲームの同時意思決定構造には不向き**。CFR/ReBeL/R-NaDは理論的に優れるが実装コストが非現実的。
